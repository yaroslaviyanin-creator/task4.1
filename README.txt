Проект: Многосвязный список строк и кастомный менеджер памяти
Автор: Янин Ярослав Иванович, группа МК-101


* Менеджер памяти (mm.c): Наш собственный аллокатор. Берет память большими кусками, чтобы не фрагментировать кучу. Удаленные элементы не отдает системе, а кидает в свой стек для переиспользования. Если память всё-таки кончается, он сам расширяет массив и вызывает коллбэк для пересчета всех сдвинувшихся адресов.
* Базовый список (list.c): Простой двусвязный кольцевой список. Узлы хранят только указатели на соседей, а сами данные в них не лежат (используем хак с макросами смещений).
* Мульти-список строк (string_list.c): Строка хранится в памяти ровно один раз, но её контейнер прошит сразу шестью независимыми кольцевыми связями. Это позволяет мгновенно получать элементы в порядке добавления, отсортированными по длине (пузырек) или лексикографически (quicksort) — в прямом и обратном порядке, без дублирования самих строк.

ПРИМЕР ЗАПУСКА:
> ./my_program.exe

Программа сама прогонит два этапа:
1. Базовый тест: закинет набор тестовых строк и выведет их на экран во всех 6-ти возможных порядках.
2. Стресс-тест памяти: в цикле сгенерирует сотни случайных строк, чтобы специально спровоцировать нехватку памяти. Менеджер расширит пул, а функция realloc_callback аккуратно пересчитает все указатели в списках[cite: 1]. Затем половина строк удалится, чтобы проверить работу стека освобожденных блоков[cite: 1].

СБОРКА И ТЕСТИРОВАНИЕ ПРОЕКТА (CMake)

1. ГЕНЕРАЦИЯ ПРОЕКТА
Создает нужные файлы для сборки в отдельной папке build. 
Выполнять из корневой директории проекта:
> cmake -B build

(Если нужна конкретная версия студии: cmake -B build -G "Visual Studio 17 2022" -A x64)
Проект можно открыть в Visual Studio через файл CMakeLists.txt или сгенерированный .sln.

2. СБОРКА ПРОЕКТА
Компиляция исходного кода. Выполнять из корня проекта:
> cmake --build build --config Debug
(Для релизной версии замените Debug на Release)
